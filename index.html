  • Para enviar cambios locales a un repositorio remoto, ¿cuál es el comando correcto? git push

    • Git push envía los cambios del repositorio local al repositorio en Git, siempre que esten commiteados.

  • ¿Cuál es el propósito del comando "git pull"? Descargar y fusionar cambios del repositorio remoto

    • A diferencia del push, hace lo contrario y descarga el repositorio de Git

  • ¿Qué comando se utiliza para descargar un repositorio remoto en tu máquina local? git clone

    • A diferencia del pull, solo los descarga, sin tener en cuenta cambios o modificaciones.

  • ¿Cómo puedes verificar los cambios realizados en una solicitud de extracción (pull request) en GitHub web? Revisando el historial de confirmaciones en la interfaz web de GitHub

    • Hasta donde se, la confirmación de los cambios de una pull request solo puede ser realizada en Web.

  • Si deseas fusionar una rama específica con la rama principal pero quieres mantener un historial de commits limpio, ¿cuál es el mejor enfoque? Hacer una fusión agregando al comando git merge el modificador --squash

    • El modificador squash unifica todos los commits en uno, y lo añade a la rama elegida.

  • Después de resolver manualmente los conflictos entre dos ramas con un conflicto de fusión, ¿cuál es el siguiente paso para completar la fusión en Git? Git push para aplicar los cambios resueltos.

    • Commitear implicaría tratar los cambios más alla del merge, es la mejor opción.

  • ¿Cuál es la función principal de la pestaña "Source Control" en Visual Studio Code cuando trabajas con Git? Visualizar y gestionar cambios en el código fuente.

    • Permite tener un registro de los añadidos y commits realizados de forma visual.

  • ¿Es posible instalar y utilizar Git en sistemas operativos Windows? Sí, Git es compatible con Windows y puede instalarse y utilizarse desde la línea de comandos.

    • Gracias a una extensión, se puede utilizar una consola Bash en Windows, y por tanto, Git.
